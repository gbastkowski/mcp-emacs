# Plan: Emacs Bootstrap Without Installed Package

## Goal
Enable the MCP server to run without requiring the Emacs package to be installed.
The server should always evaluate a bootstrap Elisp payload via `emacsclient --eval` when starting, without attempting `(require 'mcp-emacs)`.
`server-start` and `emacsclient` remain required.

## Recommended Approach
1) Generate a bootstrap Elisp payload at build time from `packages/emacs/lisp/mcp-emacs.el`.
2) Embed that payload in the server bundle and always evaluate it at server startup.
3) Keep the Emacs package available as an optional install, but do not rely on `(require 'mcp-emacs)`.

## Planned Changes
- `packages/server/src/emacs-client.ts`
  - Change `ensureInitialized()` to:
    - Evaluate the generated bootstrap Elisp payload unconditionally.
    - Mark `initialized` true after successful bootstrap.
  - Keep error message for Emacs server / emacsclient failures unchanged.

- `packages/server/src/bootstrap-elisp.ts` (new)
  - Export a string payload containing the Elisp definitions (generated at build time).
  - The payload should end with `(provide 'mcp-emacs)`.

- `packages/server/scripts/extract-elisp.ts` (new)
  - Read `packages/emacs/lisp/mcp-emacs.el`.
  - Extract the full file contents as a string literal (no transformation beyond escaping).
  - Write `packages/server/src/bootstrap-elisp.ts`.

- `packages/server/package.json`
  - Add a build step to run the extractor before `tsc` (e.g., `"build": "node scripts/extract-elisp.ts && tsc"`).

- `packages/emacs/lisp/mcp-emacs.el`
  - No functional change required.
  - If needed for clarity, add a short comment near `(provide 'mcp-emacs)` noting it is also used by the server bootstrap.

- `README.md` and `packages/emacs/README.md`
  - Update setup docs to explain that the server self-bootstraps without requiring the Emacs package.
  - Keep the package install path as optional.

## Notes / Constraints
- The bootstrap payload must be ASCII-only.
- The fallback should be automatic.
- Emacs server must already be running; no attempt to start it from the server.

## Verification
1) With package installed:
   - Start Emacs server.
   - Run `npm run build` in `packages/server`.
   - Start MCP server and verify tools work (e.g. get buffer content, insert at point).
2) Without package installed:
   - Remove `mcp-emacs` from `load-path`.
   - Start Emacs server.
   - Start MCP server and verify the same tools work.
3) Error cases:
   - Stop Emacs server and confirm error remains clear.
   - Rename `emacsclient` in PATH and confirm error remains clear.
